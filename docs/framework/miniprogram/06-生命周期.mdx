# 生命周期

## 小程序运行机制

<img src={workWay} alt="运行机制" style={{zoom: '50%'}} />

### 冷启动与热启动

小程序启动可以分为“冷启动”和“热启动”两种情况。

- 冷启动：用户首次打开小程序，或小程序被销毁后再次打开，此时小程序需要重新加载启动。
- 热启动：用户已经打开过小程序，在一定时间内再次打开，此时小程序未被销毁，只是从后台切换到前台。

### 前台与后台状态

小程序启动后，界面展示给用户，此时小程序处于“前台”状态。

当用户关闭小程序时，小程序并没有真正被关闭，而是进入了“后台”状态，当用户再次打开该小程序，小程序又会重新进入“前台”状态。

切后台的方式包括但不限于以下几种：

- 点击右上角胶囊关闭按钮离开小程序。
- 安卓点击返回键离开小程序。
- 屏幕左侧右滑离开小程序。

### 挂起与销毁

**挂起**

小程序进入后台一段时间后（5 秒），微信停止小程序 JS 线程的执行，小程序进入“挂起”状态。

:::caution
当小程序使用了后台播放音乐、后台地理位置等能力时，小程序可以在后台持续运行，不会进入“挂起”状态。
:::

**销毁**

当小程序挂起 30 分钟后，或者系统资源紧张，小程序就会被销毁，即完全终止运行。

- 当小程序进入后台并被“挂起”后，30 分钟内都未被再次使用，小程序会被销毁。
- 当小程序占用系统资源过高，5 秒内连续两次出现内存告警，小程序也会被销毁。

## 小程序更新机制

访问小程序时，微信会将小程序代码包缓存到本地。当小程序有新版本发布时，微信会自动下载更新的小程序代码包，并覆盖本地缓存的小程序代码包。

:::tip
- 如果更新失败，还是会使用本地版本。
- 在小程序发布 24 小时后，基本上能覆盖 99% 以上的用户。
- 小程序管理后台也可以设置“优先使用本地版本”和“小程序最低可用版本”。
:::

小程序的更新机制有两种：启动时同步更新 和 启动时异步更新。

**启动时同步更新**

- 对于最近使用的小程序，微信会定期检查这些小程序是否有更新，如果有更新，就更新小程序代码包。在打开这些小程序时，会同步更新到最新版本再启动。
- 对于长期未使用的小程序，在打开小程序时，微信会强制同步检查版本更新，更新到最新版本后再打开小程序。

**启动时异步更新**

在启动小程序之前，客户端没有发现小程序有新版本，小程序在每次冷启动时，也会异步检查是否有更新版本。如果有新版本，就会异步下载新版本代码包，在下一次冷启动时使用新版本小程序，本次仍然使用旧版本。

如果希望立即使用新版本，开发者可以手动调用 `wx.getUpdateManager`，提示用户重启小程序更新版本。

```js
App({
  // 当小程序冷启动初始化完成时，会触发 onLaunch（全局只触发一次）
  onLaunch() {
    // 小程序冷启动时，会自动向微信后台请求新版本信息，如果有新版本，会自动下载新版本代码包
    // 通过 wx.getUpdateManager() 方法可以监听下载状态
    const updateManager = wx.getUpdateManager()

    // 请求完新版本信息的回调
    updateManager.onCheckForUpdate(function (res) {
      console.log(res.hasUpdate)
    })

    // 新版本下载完成后，会触发 onUpdateReady
    updateManager.onUpdateReady(function () {
      wx.showModal({
        title: '更新提示',
        content: '新版本已经准备好，是否重启应用？',
        success(res) {
          if (res.confirm) {
            // 新的版本已经下载好，调用 applyUpdate 应用新版本并重启
            updateManager.applyUpdate()
          }
        }
      })
    })

    // 新版本下载失败
    updateManager.onUpdateFailed(function () {
      // ...
    })
  }
})
```

配置“编译模式”时，可以选择模拟更新到新版本。

## 生命周期

小程序的生命周期分为三类：应用生命周期、页面生命周期、组件生命周期。

### 应用生命周期

应用生命周期函数需要在 app.js 文件的 `App()` 方法中定义。

`App()` 方法用来注册小程序应用，必须在 app.js 中调用，且只能调用一次。

```js title="app.js"
App({
  // 当小程序初始化完成时，会触发 onLaunch（全局只触发一次）
  // 当进行冷启动时，才会触发 onLaunch 钩子
  // 如果是热启动，不会触发 onLaunch，而是触发 onShow
  // 所以 onLaunch 全局只会触发一次
  onLaunch() {
    console.log('onLaunch: 当小程序初始化完成时，会触发 onLaunch')
  },

  // 当小程序启动，或从后台切换到前台，会触发 onShow
  onShow(options) {
    console.log('onShow: 当小程序启动，或从后台进入前台显示')
  },

  // 当小程序从前台切换到后台，会触发 onHide
  onHide() {
    console.log('onHide: 小程序从前台进入后台')
  }
})
```

### 页面生命周期

页面生命周期函数需要在页面的 .js 文件的 `Page()` 方法中定义。

```js
Page({
  // 页面加载时调用，一个页面只会调用一次
  onLoad(options) {
    console.log('页面加载完毕')
  },

  // 页面显示、后台切前台时调用
  onShow() {
    console.log('监听页面显示，此时页面处于显示状态')
  },

  // 页面初次渲染完成调用，一个页面只会调用一次
  // 代表页面已经准备妥当，可以和视图层进行交互
  onReady() {
    console.log('页面初次渲染已经完成')
  },

  // 页面隐藏（如 navigate）、前台切后台时调用
  onHide() {
    console.log('当前页面处于隐藏状态')
  },

  // 页面卸载时调用（如 redirectTo）
  onUnload() {
    console.log('页面卸载时触发')
  }
})
```

### 两个细节

- 点击 tabBar 进行页面间的切换，页面不会被销毁。
- 点击左上角返回按钮，返回上一个页面，当前页面会被销毁。
