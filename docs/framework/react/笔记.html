笔记

8. 函数组件的状态​

函数组件通常只考虑负责 UI 的渲染，没有自身状态和业务逻辑代码，是一个纯函数。

const Welcome = (props) => {
  const [num, setNum] = useState(0);
  const [str, setStr] = useState("hello");
  let fn = () => {
    setNum(100);
  };
  let fn2 = () => {
    setStr("world");
  };
  return (
    <div>
      我是app {num}
      <h1>{str}</h1>
      <button onClick={fn}>改num</button>
      <button onClick={fn2}>改str</button>
    </div>
  );
};

21. Hooks​

Hooks 是 React 16.8 的新增特性，允许你在不编写 class 的情况下使用 state 以及其他的 React 特性。

22. Hooks 的使用​

Hooks 是基于函数组件开始设计，所以 Hooks 只支持函数组件！！！ Hooks 提供了类似类组件中 state 和生命周期的一些功能，但更加灵活。

const [count, setCount] = useState(0);
useEffect(() => {
  console.log("组件挂载了");
}, []); // 空数组表示不依赖任何变量，只会在组件挂载时执行

23. useState​

useState 是用来给函数组件提供 state 的 Hook。 基本语法：

const [state, setState] = useState(initialState);

useState 返回的更新对象的函数是异步的，要在下次重绘才能获取新值，不要试图在更改状态之后立即获取状态。

24. useEffect​

useEffect 是用来处理副作用的 Hook，例如数据获取、订阅以及手动更改 DOM。 基本语法：

useEffect(() => {
  // 会在组件挂载和更新时执行
}, [依赖项]); // 依赖项数组，只有当数组中的依赖项改变时，才会重新执行

useEffect 可以接收第二个参数，为空数组时表示不依赖任何变量，只会在组件挂载时执行。

25. useContext​

useContext 是用来读取 context 的值以及订阅 context 的变化的 Hook。 基本用法：

const MyContext = React.createContext("");
const MyComponent = () => {
  const value = useContext(MyContext);
  return <div>{value}</div>;
};

26. React.memo​

React.memo 是一个高阶组件（HOC），用于对函数组件进行性能优化。 基本用法：

const MyComponent = React.memo(({ prop1, prop2 }) => {
  // 渲染逻辑
});

React.memo 仅检查组件的 props 变化，如果 props 没有改变，则不会重新渲染该组件。

27. useMemo​

useMemo 是用来缓存数据的 Hook，只有当依赖项改变时才会重新计算。 基本用法：

const memoizedValue = useMemo(() => computeExpensiveValue(a, b), [a, b]);

28. useCallback​

useCallback 是用来缓存函数的 Hook，只有当依赖项改变时才会重新创建函数实例。 基本用法：

const memoizedCallback = useCallback(() => doSomething(a, b), [a, b]);

29. useReducer​

useReducer 是用来处理更复杂的组件状态逻辑的 Hook。 基本用法：

const [state, dispatch] = useReducer(reducer, initialArg, init);

useReducer 接受两个参数：一个 reducer 函数和一个初始状态（initial state）。

30. useRef​

useRef 是用来在函数组件中创建引用（ref）对象的 Hook。 基本用法：

const myRef = useRef();

useRef 可以用来访问 DOM 元素或存储可变值。

31. useId​

useId 是用来生成唯一 ID 字符串的 Hook。 基本用法：

const id = useId();

useId 在服务端渲染（SSR）和客户端渲染中保持一致。

32. useDeferredValue​

useDeferredValue 是用来延迟单个值更新的 Hook。 基本用法：

const deferredValue = useDeferredValue(initialValue);

useDeferredValue 主要用于延迟不那么重要的更新，不会影响用户的输入体验。

33. useTransition​

useTransition 是用来标记低优先级状态更新的 Hook。 基本用法：

const [isPending, startTransition] = useTransition();

useTransition 允许你在触发更新时提供更自然的用户体验。

34. useImperativeHandle​

useImperativeHandle 是用来在函数组件中暴露方法和属性的 Hook。 基本用法：

useImperativeHandle(ref, () => ({
  method: () => doSomething(),
}));

useImperativeHandle 结合 forwardRef 使用，允许父组件访问子组件的方法或属性。

35. useLayoutEffect​

useLayoutEffect 是用来在浏览器绘制之前同步执行的副作用 Hook。 基本用法：

useLayoutEffect(() => {
  // 会在浏览器绘制之前执行
}, [依赖项]);

useLayoutEffect 可以用来读取 DOM 布局并在浏览器绘制之前立即同步更改它。

36. 自定义 Hooks​

自定义 Hooks 是用来将组件逻辑提取到可重用函数中的模式。 基本规则：

  • 自定义 Hooks 的名称必须以 “use” 开头。
  • 自定义 Hooks 内部可以调用其他的 Hooks。

37. React-Router​

React-Router 是用来管理 React 应用的路由的库。 基本用法：

import { BrowserRouter as Router, Route, Switch } from 'react-router-dom';
import Home from './Home';
import About from './About';
function App() {
  return (
    <Router>
      <Switch>
        <Route path="/" exact component={Home} />
        <Route path="/about" component={About} />
      </Switch>
    </Router>
  );
}

38. 路由的跳转​

  • 标签跳转：使用 <Link> 组件。
  • 编程式导航：使用 useNavigate 钩子。
  • 路由的传参：使用 useParams 和 useSearchParams 钩子。

39. 嵌套路由​

  • 使用 <Outlet> 组件来渲染嵌套路由。
  • 使用 React.lazy 和 Suspense 来实现路由的懒加载。

40. 路由模式​

  • createBrowserRouter：使用 HTML5 History API。
  • createHashRouter：使用 URL 的 hash 部分。

41. Data API​

  • createApi：创建 API 配置。
  • fetchBaseQuery：配置基础路径。
  • endpoints：配置请求的详细信息。

42. RTK Query​

  • useGetListQuery：获取列表数据。
  • useAddUserMutation：添加用户数据。
  • useUpdateUserMutation：更新用户数据。
  • useDeleteUserMutation：删除用户数据。

43. 异步逻辑与数据请求​

  • 使用 createAsyncThunk 来处理 Redux 中的异步操作。
  • 使用 extraReducers 来处理在 slice 外部定义的 actions。
  • 使用 createSelector 来从 Redux 的全局状态中导出一些衍生数据。

44. 缓存操作​

  • 使用 transformResponse 来转换响应数据。
  • 使用 invalidatesTags 和 providesTags 来控制缓存。

45. 总结​

React 18 和 Hooks 为开发者提供了更加强大和灵活的工具来构建用户界面。 通过使用 Hooks 和 RTK Query，可以简化状态管理和数据请求的逻辑。 React-Router 允许你创建动态和响应式的应用程序。 使用 React.lazy 和 Suspense 可以实现路由的懒加载，提高应用程序的性能。 Data API 提供了管理和预加载数据的功能。 通过使用这些工具和库，可以构建更高效、可维护和响应式的应用程序。
